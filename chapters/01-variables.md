# Глава 1. Переменные и типы данных

> **Цель главы:** глубоко понять, что такое переменная, какие типы данных есть в JS, чем `let`/`const`/`var` отличаются, как работает преобразование типов, и какие подводные камни скрывает динамическая типизация.

---

## 1. Что такое переменная

**Переменная** — это именованная ячейка памяти, в которой хранится значение. Представь шкафчик в раздевалке: на шкафчике табличка с именем (`age`), внутри лежит содержимое (`25`). Ты можешь заглянуть внутрь, заменить содержимое, но имя шкафчика остаётся.

```js
let age = 25;        // положили 25 в шкафчик "age"
console.log(age);    // заглянули → 25

age = 26;            // заменили содержимое на 26
console.log(age);    // → 26
```

**Технически:** при объявлении переменной JS-движок выделяет участок памяти и связывает с именем. Имя — это метка, по которой движок находит значение в памяти.

### Анатомия объявления

```js
let username = "Дана";
```

Разбор по частям:
- `let` — **ключевое слово**, говорит движку «создай переменную»
- `username` — **имя переменной** (идентификатор)
- `=` — **оператор присваивания**, кладёт значение в переменную
- `"Дана"` — **значение**
- `;` — конец инструкции (ставь всегда, даже если JS «прощает» её отсутствие)

## 2. Три способа объявить переменную

В JS есть три ключевых слова: `let`, `const`, `var`. В современном коде используют первые два. `var` — историческое наследие, которое почти не применяется.

### `let` — можно переприсваивать

```js
let score = 0;
score = 10;          // OK
score = score + 5;   // OK
console.log(score);  // 15
```

Используй `let`, когда значение **будет меняться** во время работы программы.

**Блочная область видимости:** переменная `let` живёт только внутри фигурных скобок `{}`, где объявлена.

```js
{
  let secret = 42;
  console.log(secret); // 42 — внутри блока видно
}
console.log(secret);   // ❌ ReferenceError: secret is not defined
```

### `const` — нельзя переприсваивать

```js
const PI = 3.14159;
PI = 3;               // ❌ TypeError: Assignment to constant variable.
```

**Важный нюанс:** `const` делает неизменяемой **ссылку**, а не содержимое. Для объектов и массивов это значит:

```js
const user = { name: "Дана" };
user.name = "Иван";    // ✅ OK! Содержимое менять можно
user.age = 25;         // ✅ OK! Добавили новое свойство
user = { name: "Иван" }; // ❌ Ошибка! Ссылку менять нельзя
```

Это часто путает новичков. Запомни: `const` = «нельзя перезаписать переменную», но не «нельзя менять объект внутри».

**Правило:** по умолчанию используй `const`. Меняй на `let`, только когда точно знаешь, что будешь переприсваивать. Это делает код безопаснее и показывает намерение: «это значение не должно меняться».

### `var` — устаревший, не используй

`var` — старый способ из до-2015 года (до стандарта ES6). Имеет странное поведение, которое приводит к багам.

**Особенность 1 — всплытие (hoisting):**
```js
console.log(x);  // undefined (не ошибка!)
var x = 5;
```
`var` «всплывает» наверх области видимости, но без значения. Поэтому `undefined`, а не ошибка. Это скрывает баги.

**Особенность 2 — функциональная (не блочная) область видимости:**
```js
if (true) {
  var leak = "утёк наружу";
}
console.log(leak);  // "утёк наружу" — var проигнорировал блок if!
```
С `let` такой утечки нет — переменная осталась бы внутри блока.

**Вывод:** никогда не используй `var`. Весь современный код на `let`/`const`.

### Шпаргалка

| | `let` | `const` | `var` |
|---|---|---|---|
| Переприсваивание | ✅ | ❌ | ✅ |
| Область видимости | блочная | блочная | функциональная |
| Всплытие (hoisting) | TDZ (ошибка) | TDZ (ошибка) | undefined |
| Использовать? | когда меняется | **по умолчанию** | ❌ никогда |

## 3. Temporal Dead Zone (TDZ)

Это понятие пугает новичков, но на деле простое. **TDZ** — это зона между началом блока и объявлением переменной, где к ней нельзя обратиться.

```js
{
  // ↓↓↓ Здесь начинается TDZ для age ↓↓↓
  console.log(age);  // ❌ ReferenceError
  // ↑↑↑ TDZ продолжается ↑↑↑
  let age = 25;      // ← конец TDZ
  console.log(age);  // ✅ 25
}
```

С `var` такой ошибки нет (получишь `undefined`), что скрывает баги. `let`/`const` честно говорят: «ты объявил переменную ниже, а обращаешься раньше — так нельзя».

---

## 4. Правила именования

- Только буквы, цифры, `$` и `_`. Пробелы и дефисы нельзя.
- Не могут начинаться с цифры. `let 2cool` → ошибка.
- Чувствительны к регистру: `age` и `Age` — разные переменные.
- Нельзя использовать зарезервированные слова: `let`, `return`, `function`, `class` и т.д.

**Соглашения (не правила, но все так делают):**
- **camelCase** для переменных и функций: `userName`, `totalCount`, `streakLength`
- **UPPER_SNAKE_CASE** для констант: `MAX_RETRIES`, `API_URL`
- **Описательные имена**: `userAge` хорошо, `a` или `x` — плохо (кроме счётчиков в циклах)

## 5. Типы данных

«Тип» — это **вид значения**. JS должен знать, число внутри или текст, потому что операции с ними разные.

Пример, почему типы критичны:
```js
2 + 3        // → 5      (число + число = сумма)
"2" + "3"    // → "23"   (строка + строка = склейка!)
"5" - 2      // → 3      (минус → JS понял: математика)
```

JS делит типы на **примитивные** (простые, передаются по значению) и **объекты** (сложные, передаются по ссылке). В этой главе — примитивы.

### 5.1 Number (число)

Один тип для всех чисел: целые, дробные, отрицательные.

```js
const age = 25;
const temp = -5.7;
const billion = 1_000_000_000;   // подчёркивания для читаемости
const large = 9e9;               // экспонента = 9 миллиардов
```

**Специальные числовые значения:**
```js
1 / 0          // → Infinity
-1 / 0         // → -Infinity
0 / 0          // → NaN (Not a Number)
"текст" / 2    // → NaN (не удалось вычислить)
```

**NaN — особенный:** он не равен ничему, даже себе!
```js
NaN === NaN    // → false (!!!)
Number.isNaN(NaN)  // → true (правильная проверка)
```

**Предел точности:** JS хранит числа как числа с плавающей точкой (IEEE 754). Безопасный диапазон целых: от `Number.MIN_SAFE_INTEGER` (-2^53+1) до `Number.MAX_SAFE_INTEGER` (2^53-1). Для больших чисел используй `BigInt`.

```js
Number.MAX_SAFE_INTEGER    // 9007199254740991
9007199254740991 + 1       // 9007199254740992
9007199254740991 + 2       // 9007199254740992 ← ОШИБКА! Точность потеряна
9007199254740991n + 2n     // 9007199254740993n ← BigInt решает
```

### 5.2 String (строка, текст)

Текст в кавычках. Три вида кавычек:

```js
const single = 'Привет';
const double = "Привет";
const backtick = `Привет`;
```

Одинарные и двойные — одинаковые. **Бэктиги (обратные кавычки)** особенные — позволяют вставлять переменные и переносы:

```js
const name = "Дана";
const msg = `Привет, ${name}!
Сегодня ${new Date().toLocaleDateString()}`;
```

**Полезные методы строк** (строки иммутабельны — методы возвращают новые строки):
```js
"Hello".length           // 5 — длина
"Hello".toUpperCase()    // "HELLO"
"Hello".toLowerCase()    // "hello"
"Hello".charAt(0)        // "H" — символ по индексу
"Hello".includes("ell")  // true — содержит ли
"Hello".indexOf("l")     // 2 — позиция первого вхождения
"Hello".slice(1, 4)      // "ell" — вырезать с 1 по 4 (не включая)
"Hello".replace("l", "L")// "HeLlo" — заменить первое
"Hello".split("")        // ["H","e","l","l","o"] — разбить в массив
"  hi  ".trim()          // "hi" — убрать пробелы по краям
```

### 5.3 Boolean (логический тип)

Два значения: `true` (истина) или `false` (ложь). Используется в условиях.

```js
const isLoggedIn = true;
const hasError = false;
```

**Что JS считает «ложноподобным» (falsy):**
```js
false        // само false
0            // ноль
-0           // минус-ноль
0n           // BigInt-ноль
""           // пустая строка
null         // null
undefined    // undefined
NaN          // NaN
```

**Всё остальное — «истинноподобное» (truthy).** Даже:
```js
"0"          // строка с нулём → truthy (не пустая!)
"false"      // строка "false" → truthy (не пустая!)
[]           // пустой массив → truthy
{}           // пустой объект → truthy
```

Это частая ловушка. Запомни: `0` falsy, но `"0"` truthy.

### 5.4 null

**Намеренное «пусто».** Программист сам ставит `null`, чтобы сказать «здесь осознанно ничего нет».

```js
let selectedUser = null;   // пользователь ещё не выбран, но мы об этом знаем
selectedUser = { name: "Дана" };  // выбрали
```

### 5.5 undefined

**«Не задано».** JS даёт это значение переменным, которым не присвоили значение.

```js
let x;                  // объявили, но не задали
console.log(x);         // undefined

function f() {}         // функция ничего не возвращает
console.log(f());       // undefined
```

**Разница null vs undefined (частый вопрос):**
- `null` — ты намеренно сказал «пусто»
- `undefined` — система сказала «ты не задал значение»
- `null === undefined` → `false`
- `null == undefined` → `true` (но `==` вообще использовать не стоит, см. главу 2)

### 5.6 Symbol (продвинутое)

Уникальный, неизменяемый идентификатор. Используется редко, в основном для скрытых свойств объектов.

```js
const id = Symbol("id");
const obj = { [id]: 123, name: "Дана" };
```

На старте можешь пропустить — просто знай, что существует.

### 5.7 BigInt (для гигантских чисел)

Когда числа больше `Number.MAX_SAFE_INTEGER`. Создаётся добавлением `n` к числу или через `BigInt()`.

```js
const huge = 9007199254740993n;
const also = BigInt("9007199254740993");
```

Нельзя смешивать `BigInt` и обычные `Number` в одной операции — будет ошибка.

## 6. Как узнать тип: typeof

Оператор `typeof` возвращает имя типа строкой:

```js
typeof 25            // "number"
typeof "Дана"        // "string"
typeof true          // "boolean"
typeof undefined     // "undefined"
typeof 123n          // "bigint"
typeof Symbol()      // "symbol"
typeof null          // "object"  ⚠️ известный баг JS!
typeof {}            // "object"
typeof []            // "object"  (массивы — это объекты!)
typeof function(){}  // "function"
```

> ⚠️ `typeof null === "object"` — исторический баг из первых версий JS. Не могут исправить, чтобы не сломать существующий код. Для проверки на null используй `value === null`.

> ⚠️ `typeof [] === "object"` — массивы технически объекты. Для проверки на массив используй `Array.isArray(value)`.

## 7. Динамическая типизация

JS — **динамически типизированный язык**. Переменная может менять тип во время выполнения:

```js
let x = 5;         // number
x = "пять";        // string — OK для JS
x = true;          // boolean — тоже OK
```

В статически типизированных языках (TypeScript, Java, C++) так делать нельзя — тип фиксируется при объявлении.

**Плюсы динамической типизации:** гибкость, быстрый старт, меньше кода.
**Минусы:** баги обнаруживаются только при выполнении, сложнее в больших проектах.

TypeScript решает эту проблему, добавляя статическую проверку типов на этапе разработки (см. отдельную главу).

## 8. Преобразование типов

Часто нужно превратить значение одного типа в другой. Есть два способа: явный и неявный.

### 8.1 Явное преобразование

Ты сам говоришь «преврати»:

```js
String(25)           // "25"
Number("123")        // 123
Number("123.45")     // 123.45
Number("")           // 0
Number("  ")         // 0
Number("abc")        // NaN (не удалось)
Number(true)         // 1
Number(false)        // 0
Number(null)         // 0
Number(undefined)    // NaN

Boolean(0)           // false
Boolean("")          // false
Boolean("текст")     // true
Boolean(null)        // false
Boolean([])          // true
Boolean({})          // true

parseInt("42px")     // 42 (целое)
parseFloat("3.14em") // 3.14 (дробное)
+"42"                // 42 (унарный плюс = быстрый Number)
```

### 8.2 Неявное преобразование

JS сам решает за тебя. Это источник большинства багов:

```js
"5" + 2      // "52"  (плюс → склейка, 2 стало "2")
"5" - 2      // 3     (минус → математика, "5" стало 5)
"5" * "2"    // 10    (умножение → математика)
"10" / "2"   // 5
1 + "2" + 3  // "123" (слева направо: 1+"2"="12", "12"+3="123")
true + 1     // 2     (true → 1)
false + "1"  // "false1"
null + 1     // 1     (null → 0)
undefined + 1 // NaN  (undefined → NaN)
```

**Правило:** никогда не полагайся на неявное преобразование. Если нужна математика — явно приведи к числу через `Number()`. Если склейка строк — через `String()` или шаблонные строки. Код станет предсказуемым.

## 9. Сравнение значений: == vs ===

Это **самый частый источник багов** у новичков.

- `==` (нестрогое) — приводит типы перед сравнением
- `===` (строгое) — не приводит, сравнивает как есть

**Таблица неочевидных результатов `==`:**

| Выражение | Результат | Почему |
|-----------|-----------|--------|
| `"" == false` | `true` | оба → 0 |
| `"0" == false` | `true` | оба → 0 |
| `"0" == 0` | `true` | строка → число |
| `null == undefined` | `true` | специальное правило |
| `null == 0` | `false` | исключение для null |
| `NaN == NaN` | `false` | NaN не равен ничему |
| `[1] == 1` | `true` | массив → "1" → 1 |
| `[] == false` | `true` | массив → "" → 0 |

С `===` все эти сравнения дадут `false`, потому что типы разные. **Всегда используй `===`.**

`Object.is()` — третье сравнение, решает два edge-case'а:
```js
Object.is(NaN, NaN)     // true  (NaN === NaN даёт false)
Object.is(-0, 0)        // false (-0 === 0 даёт true)
Object.is("a", "a")     // true  (как ===)
```

## 10. Частые ошибки новичков

### Ошибка 1: путают `=` и `===` в условиях
```js
if (x = 5) { ... }    // ❌ присваивание, всегда true
if (x === 5) { ... }  // ✅ сравнение
```

### Ошибка 2: пытаются изменить `const`
```js
const arr = [1, 2, 3];
arr = [4, 5];         // ❌ TypeError
arr.push(4);          // ✅ содержимое менять можно
```

### Ошибка 3: думают, что `"0"` — falsy
```js
if ("0") { console.log("выведется!"); }  // выведется! "0" — truthy
```

### Ошибка 4: забывают про NaN
```js
NaN === NaN           // false
Number("abc") === NaN // false (!!!) — используй Number.isNaN()
```

### Ошибка 5: неявное сложение строк и чисел
```js
"Цена: " + 100 + "₽"  // "Цена: 100₽" — OK
100 + 50 + "₽"        // "150₽" — OK (сначала математика)
"₽" + 100 + 50        // "₽10050" — баг! (слева направо)
```

### Ошибка 6: `typeof null`
```js
typeof null === "null"  // false
typeof null === "object" // true (баг JS)
```

### Ошибка 7: теряют точность с большими числами
```js
9999999999999999 === 10000000000000000  // true! точность потеряна
```

### Ошибка 8: null vs undefined путаница
```js
null == undefined    // true (но это разные вещи)
null === undefined   // false (правильно)
```

### Ошибка 9: используют `var`
```js
for (var i = 0; i < 3; i++) { ... }
console.log(i);      // 3 — var утёк из цикла! (с let был бы ReferenceError)
```

### Ошибка 10: забывают, что массивы — ссылочные
```js
const a = [1, 2, 3];
const b = a;
b.push(4);
console.log(a);      // [1, 2, 3, 4] — изменился и a! (подробно в главе 7)
```
