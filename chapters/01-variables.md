# Глава 1. Переменные и типы данных

> **Цель главы:** понять, что такое переменная, какие типы данных есть в JS, и чем `let`/`const`/`var` отличаются.

---

## 1. Что такое переменная

**Переменная** — это «коробка с именем», в которой хранится значение. В любой момент можно посмотреть, что внутри, или положить что-то новое.

```js
let age = 25;        // коробка с именем "age", внутри — число 25
console.log(age);    // заглянули в коробку → 25

age = 26;            // заменили содержимое
console.log(age);    // → 26
```

**Аналогия:** представь шкафчик в раздевалке. На шкафчике табличка `age`, внутри лежит 25. Ты можешь выложить содержимое и заменить на другое. Имя шкафчика остаётся.

### Как создать переменную (объявить)

Нужно два слова: ключевое слово + имя.

```js
let username = "Дана";   // ключевое слово let, имя — username
```

**Правила именования:**
- Только буквы, цифры, `$` и `_`. Пробелы нельзя.
- Не начинаются с цифры. `let 2cool` — ошибка.
- Чувствительны к регистру: `age` и `Age` — разные переменные.
- camelCase — стандарт в JS: `userName`, `totalCount`, `streakLength`.

## 2. Три способа объявить переменную: let / const / var

В JS есть три ключевых слова. В современном коде используют два, третье — почти не используют.

### `let` — можно менять

```js
let score = 0;
score = 10;      // OK, изменили
score = score + 5;
console.log(score);  // 15
```

Используй `let`, когда значение **будет меняться** во время работы программы.

### `const` — нельзя менять (константа)

```js
const PI = 3.14159;
PI = 3;          // ❌ ОШИБКА: Assignment to constant variable.
```

Используй `const`, когда значение **не должно меняться**.

> **Хорошее правило:** по умолчанию пиши `const`. Меняй на `let`, только когда точно знаешь, что будешь переприсваивать. Это делает код безопаснее — никто (и ты сам) случайно не перезапишет.

### `var` — устаревший, не используй

`var` — это старый способ из до-2015 года. Имеет странное поведение (всплытие, функциональная область видимости), которое часто приводит к багам. В современном коде `var` не используется.

```js
var x = 5;    // работает, но так не пишут с 2015 года
```

### Шпаргалка

| Ключевое слово | Можно менять? | Когда использовать |
|----------------|---------------|---------------------|
| `const`        | ❌ нет          | **по умолчанию**, для всего, что не меняется |
| `let`          | ✅ да           | когда значение будет меняться |
| `var`          | ✅ да           | никогда (устаревшее) |

## 3. Типы данных

«Тип» — это **вид значения**, которое лежит в переменной. JS должен знать, что внутри — число или текст, потому что операции с ними разные.

Пример, почему типы важны:
```js
2 + 3              // → 5 (число + число = сумма)
"2" + "3"          // → "23" (строка + строка = склейка!)
```

В JS есть **примитивные** типы (простые) и **объекты** (сложные). Разберём примитивы.

### Примитивные типы данных

#### Number (число)
Любые числа: целые, дробные, отрицательные. Один тип — для всех.
```js
const age = 25;
const temperature = -5.7;
const billion = 1_000_000_000;   // подчёркивания для читаемости
const large = 9e9;               // экспоненциальная запись = 9 миллиардов
```

#### String (строка, текст)
Текст. Обозначается кавычками — одинарными `'`, двойными `"`, или обратными `` ` `` (backtick).
```js
const name = "Дана";
const greeting = 'Привет!';
const long = `Можно делать
переносы строк`;
```

**Backticks `` ` `` позволяют вставлять переменные прямо в строку** — это называется шаблонные строки (template literals):
```js
const name = "Дана";
const msg = `Привет, ${name}!`;   // → "Привет, Дана!"
```

#### Boolean (логический тип)
Только два значения: `true` (истина) или `false` (ложь). Используется для ответов да/нет.
```js
const isLoggedIn = true;
const hasError = false;
```

#### null
«Ничего нет». Специальное значение, которое программист **намеренно** ставит, чтобы показать «здесь пусто».
```js
let selectedUser = null;   // никто не выбран, но мы об этом знаем
```

#### undefined
«Не задано». Значение, которое JS даёт переменной, которой **не присвоили значение**.
```js
let x;                     // объявили, но не задали значение
console.log(x);            // → undefined
```

#### Symbol и BigInt (продвинутое, можно пропустить в первый раз)
- **Symbol** — уникальный идентификатор, используется редко.
- **BigInt** — для очень больших чисел: `9007199254740991n`.

### Полная картина примитивов

| Тип | Пример | Что хранит |
|-----|--------|------------|
| Number | `25`, `-5.7`, `1e9` | Любые числа |
| String | `"Привет"`, `` `Hi ${name}` `` | Текст |
| Boolean | `true`, `false` | Да/нет |
| null | `null` | Намеренное «пусто» |
| undefined | `undefined` | «Не задано» автоматически |
| Symbol | `Symbol("id")` | Уникальный ID (редко) |
| BigInt | `123n` | Гигантские числа |

## 4. Как узнать тип значения

Оператор `typeof` возвращает имя типа строкой.

```js
console.log(typeof 25);           // → "number"
console.log(typeof "Дана");       // → "string"
console.log(typeof true);         // → "boolean"
console.log(typeof undefined);    // → "undefined"
console.log(typeof null);         // → "object"  ⚠️ это известный баг JS!
```

> ⚠️ `typeof null` возвращает `"object"` — это исторический баг языка, который не могут исправить, чтобы не сломать старый код. Запомни как странность, на первых порах просто знай.

## 5. Динамическая типизация

JS — **динамически типизированный язык**. Это значит, что переменная может менять тип.
```js
let x = 5;         // x — число
x = "пять";        // x — строка. ОК для JS!
x = true;          // x — булево. Тоже ОК.
```

В некоторых языках (TypeScript, Java, C++) так делать нельзя — там тип переменной фиксируется раз и навсегда. В JS — можно. Это и гибко, и опасно: можно случайно записать строку в переменную, где ждали число.

## 6. Преобразование типов

Иногда нужно превратить один тип в другой. Есть два способа: явный и неявный.

### Явное (когда ты сам говоришь «преврати»)

```js
String(25);        // → "25"  (число → строку)
Number("123");     // → 123   (строка → число)
Boolean(0);        // → false
```

### Неявное (когда JS сам решает, обычно к плохому)

```js
"5" - 2            // → 3   (JS понял: минус — это математика, превратил "5" в 5)
"5" + 2            // → "52"  (плюс — это склейка строк, превратил 2 в "2")
console.log("5" + 2)   // "52" — частый источник багов!
```

> **Совет:** никогда не полагайся на неявное преобразование. Если хочешь, чтобы строки склеились — явно используй `String()`. Если хочешь математику — явно `Number()`. Код будет предсказуемым.
