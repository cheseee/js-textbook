# Глава 4. Циклы

> **Цель главы:** научиться повторять действия много раз без копирования кода. Освоить `while`, `for`, `for...of`, `for...in`, `break`, `continue`, и понять, какой цикл когда применять.

---

## 1. Что такое цикл и зачем он нужен

**Цикл** — это конструкция, которая повторяет блок кода, пока выполняется условие.

**Аналогия:** тебе нужно помыть 10 тарелок. Вместо того чтобы писать «помой тарелку 1, помой тарелку 2, ...» 10 раз, ты говоришь: «повторяй, пока есть грязные тарелки: помой одну».

Без цикла:
```js
console.log(1);
console.log(2);
console.log(3);
// ... ещё 97 строк
```

С циклом:
```js
for (let i = 1; i <= 100; i++) {
  console.log(i);
}
```

---

## 2. `while` — пока условие истинно

Простейший цикл. Проверяет условие → если true, выполняет тело → снова проверяет.

```js
let i = 1;
while (i <= 5) {
  console.log(i);
  i++;
}
// выведет 1, 2, 3, 4, 5
```

**Опасность — бесконечный цикл:** если забыть `i++`, условие всегда true, цикл никогда не закончится, вкладка зависнет.

```js
let i = 1;
while (i <= 5) {
  console.log(i);
  // забыли i++ → БЕСКОНЕЧНЫЙ ЦИКЛ 💀
}
```

---

## 3. `do...while` — сначала делает, потом проверяет

Отличие от `while`: тело выполнится **хотя бы один раз**, даже если условие сразу false.

```js
let i = 10;
do {
  console.log(i);
  i++;
} while (i <= 5);
// выведет 10 один раз (условие false, но тело уже выполнилось)
```

Сравнение:
```js
// while — может не выполниться ни разу
let x = 10;
while (x < 5) { console.log("не выведется"); }

// do...while — выполнится хотя бы раз
let y = 10;
do { console.log("выведется один раз"); } while (y < 5);
```

**Когда использовать `do...while`:** когда нужно сначала сделать действие, а потом решить — повторять ли. Например, запросить ввод у пользователя и повторять, пока он не введёт корректные данные.

## 4. `for` — классический цикл со счётчиком

Самый частый цикл. Три части в одной строке:

```js
for (инициализация; условие; шаг) {
  // тело
}
```

```js
for (let i = 0; i < 5; i++) {
  console.log(i);
}
// 0, 1, 2, 3, 4
```

**Разбор частей:**
1. `let i = 0` — выполняется **один раз** до цикла (создаём счётчик)
2. `i < 5` — проверяется **перед каждой итерацией** (пока true — продолжаем)
3. `i++` — выполняется **после каждой итерации** (обновляем счётчик)

**Порядок выполнения:**
```
инициализация → условие → тело → шаг → условие → тело → шаг → ... → условие false → выход
```

### Любую часть можно пропустить

```js
let i = 0;
for (; i < 5; i++) { }       // пропустили инициализацию

for (let i = 0; i < 5;) {    // пропустили шаг
  console.log(i);
  i++;
}

for (;;) { }                  // бесконечный цикл (все пропущены)
```

## 5. `for...of` — современный перебор массивов и строк

Появился в ES6. Самый удобный способ перебрать массив — не нужно возиться с индексами.

```js
const fruits = ["яблоко", "банан", "вишня"];
for (const fruit of fruits) {
  console.log(fruit);
}
// яблоко, банан, вишня
```

Работает со строками:
```js
for (const char of "Привет") {
  console.log(char);
}
// П, р, и, в, е, т
```

**Когда использовать:** всегда, когда нужно перебрать массив и не нужен индекс.

---

## 6. `for...in` — перебор ключей объекта

Перебирает **ключи** объекта (не значения!).

```js
const user = { name: "Дана", age: 25, city: "Москва" };
for (const key in user) {
  console.log(key, "→", user[key]);
}
// name → Дана
// age → 25
// city → Москва
```

**⚠️ НЕ используй `for...in` для массивов!** Он перебирает ключи, а у массива ключи — это индексы (строки "0", "1", "2"...), плюс может захватить лишние свойства. Для массивов используй `for...of` или методы `forEach`/`map`.

```js
// ❌ плохо для массивов
const arr = [10, 20, 30];
for (const i in arr) {
  console.log(i);        // "0", "1", "2" (строки, не числа!)
}

// ✅ правильно
for (const val of arr) {
  console.log(val);      // 10, 20, 30
}
```

## 7. `break` и `continue`

### `break` — выйти из цикла полностью

```js
for (let i = 1; i <= 10; i++) {
  if (i === 5) break;       // на 5 выходим
  console.log(i);
}
// 1, 2, 3, 4
```

### `continue` — пропустить текущую итерацию, перейти к следующей

```js
for (let i = 1; i <= 5; i++) {
  if (i === 3) continue;    // пропускаем 3
  console.log(i);
}
// 1, 2, 4, 5
```

**Практический пример — фильтрация:**
```js
const nums = [1, 2, 3, 4, 5, 6];
for (const n of nums) {
  if (n % 2 !== 0) continue;  // пропускаем нечётные
  console.log(n);             // 2, 4, 6
}
```

## 8. Вложенные циклы

Один цикл внутри другого. Классический пример — таблица умножения:

```js
for (let i = 1; i <= 3; i++) {
  for (let j = 1; j <= 3; j++) {
    console.log(`${i} × ${j} = ${i * j}`);
  }
}
// 1×1=1, 1×2=2, 1×3=3, 2×1=2, 2×2=4, ...
```

**Важно:** внутренний цикл выполняется полностью для каждой итерации внешнего. Если внешний 3 раза, а внутренний 3 раза — всего 9 итераций. При больших размерах это дорого (O(n²)).

---

## 9. Паттерн `while(true)` с `break`

Полезен для «повторяй, пока не получишь правильный результат»:

```js
// Псевдокод: просим ввод, пока не получим число
let input;
while (true) {
  input = getInput();        // какая-то функция ввода
  if (Number.isFinite(input)) break;   // число? выходим
  console.log("Введите число!");
}
```

В браузере так делать **нельзя** (`prompt` блокирует), но в Node.js или с async-вводом — нормальный паттерн.

## 10. Частые ошибки

### Ошибка 1: забыли шаг — бесконечный цикл
```js
for (let i = 0; i < 10; ) {   // забыли i++
  console.log(i);             // 0, 0, 0, 0, ... бесконечно
}
```

### Ошибка 2: `<=` вместо `<` (off-by-one)
```js
const arr = [10, 20, 30];
for (let i = 0; i <= arr.length; i++) {   // ❌ i дойдёт до 3
  console.log(arr[i]);                     // arr[3] === undefined
}
// правильно: i < arr.length
```

### Ошибка 3: `for...in` по массиву
```js
for (const i in arr) {        // ❌ i — строка "0", "1"...
  console.log(arr[i]);        // работает, но хрупко
}
```

### Ошибка 4: `continue` после `i++` в while
```js
let i = 0;
while (i < 10) {
  if (i === 5) continue;     // ❌ i++ ниже не выполнится, бесконечный цикл на 5!
  i++;
}
// правильно: i++ до continue
```

### Ошибка 5: мутирование массива во время перебора
```js
const arr = [1, 2, 3];
for (const x of arr) {
  if (x === 2) arr.push(4);  // ❌ массив меняется во время перебора
}
```

## Шпаргалка

```js
// Классический for
for (let i = 0; i < arr.length; i++) { arr[i] }

// for...of — лучший для массивов
for (const item of arr) { item }

// for...in — только для объектов
for (const key in obj) { obj[key] }

// while — пока условие
while (cond) { }

// do...while — минимум один раз
do { } while (cond);

// break — выйти
for (const x of arr) { if (x === target) break; }

// continue — пропустить
for (const x of arr) { if (x < 0) continue; /* обработка */ }

// Какой цикл выбирать:
// массив без индекса → for...of
// нужен индекс → for (let i...)
// объект → for...in или Object.keys()
// «пока не...» → while
``
