# Глава 5. Функции

> **Цель главы:** понять, что такое функция, освоить все способы её создания, разобраться с замыканиями, колбэками и стрелочными функциями. Это одна из важнейших глав — функции используются везде.

---

## 1. Что такое функция

**Функция** — это именованный блок кода, который можно вызвать (выполнить) сколько угодно раз.

**Аналогия:** рецепт приготовления блюда. Ты один раз записал шаги (объявил функцию), а потом готовишь по этому рецепту много раз (вызываешь функцию), каждый раз с разными ингредиентами (аргументами).

```js
// Объявили функцию
function greet(name) {
  console.log(`Привет, ${name}!`);
}

// Вызываем несколько раз с разными аргументами
greet("Дана");    // Привет, Дана!
greet("Иван");    // Привет, Иван!
```

### Зачем нужны функции

- **DRY (Don't Repeat Yourself)** — не дублировать код. Написал один раз — используешь много.
- **Читаемость** — код разбит на именованные блоки, понятные по названию.
- **Тестирование** — функцию можно проверить отдельно.
- **Модульность** — сложную задачу разбиваем на маленькие функции.

---

## 2. Два способа создать функцию

### Способ 1: Function Declaration (объявление)

```js
function add(a, b) {
  return a + b;
}
```

### Способ 2: Function Expression (выражение)

```js
const add = function(a, b) {
  return a + b;
};
```

### В чём разница — всплытие (hoisting)

**Function Declaration** «всплывает» — можно вызвать **до** объявления:

```js
sayHi();              // ✅ работает!

function sayHi() {
  console.log("Привет!");
}
```

**Function Expression** НЕ всплывает — переменная `add` существует, но `undefined` до присваивания:

```js
sayHi();              // ❌ ReferenceError: Cannot access 'sayHi' before initialization

const sayHi = function() {
  console.log("Привет!");
};
```

**Правило:** используй **Function Declaration** для функций верхнего уровня (всплывание удобно), и **Expression** когда передаёшь функцию как значение.

## 3. Стрелочные функции (Arrow Functions)

Современный компактный синтаксис (ES6). Очень популярный.

```js
// Полная форма
const add = (a, b) => {
  return a + b;
};

// Краткая форма (implicit return) — когда тело состоит из одного выражения
const add = (a, b) => a + b;

// Один параметр — скобки можно опустить
const square = x => x * x;

// Без параметров — нужны пустые скобки
const sayHi = () => console.log("Hi");
```

### Полный и краткий синтаксис

```js
// Полная форма (тело в фигурных скобках, нужен return)
const greet = (name) => {
  return `Привет, ${name}`;
};

// Краткая форма (неявный return, без скобок)
const greet = name => `Привет, ${name}`;

// Возвращаем объект — НУЖНЫ скобки вокруг объекта!
const makeUser = (name) => ({ name: name, age: 25 });
//  ↑ без ( ) JS подумает, что это тело функции
```

### Важное отличие от обычных функций

Стрелочные функции **не имеют своего `this`** — берут `this` из внешнего контекста. Это часто полезно (особенно в колбэках), но иногда неожиданно.

```js
const obj = {
  name: "Дана",
  // Обычная функция — this указывает на obj
  greetRegular: function() {
    console.log(this.name);    // "Дана"
  },
  // Стрелочная — this берётся снаружи (undefined в strict mode)
  greetArrow: () => {
    console.log(this.name);    // undefined или ошибка
  }
};
```

Также стрелочные функции:
- Не имеют `arguments` (объект со всеми аргументами)
- Нельзя использовать как конструктор (с `new`)

## 4. Параметры и аргументы

Термины часто путают, но они разные:

- **Параметры** — переменные в объявлении функции `(a, b)`
- **Аргументы** — значения, которые ты передаёшь при вызове `add(2, 3)`

```js
function add(a, b) {   // a и b — параметры
  return a + b;
}
add(2, 3);             // 2 и 3 — аргументы
```

---

## 5. Значения по умолчанию (default parameters)

Если аргумент не передали — будет `undefined`. Можно задать значение по умолчанию:

```js
function greet(name = "Гость") {
  console.log(`Привет, ${name}!`);
}
greet("Дана");   // Привет, Дана!
greet();         // Привет, Гость!  (значение по умолчанию)
```

---

## 6. Rest-параметры (`...args`)

Собирают все оставшиеся аргументы в массив:

```js
function sumAll(...numbers) {
  return numbers.reduce((total, n) => total + n, 0);
}
sumAll(1, 2, 3);        // 6
sumAll(1, 2, 3, 4, 5);  // 15
```

---

## 7. Spread-оператор (`...arr`)

Делает противоположное rest — «распаковывает» массив в отдельные значения:

```js
const nums = [1, 2, 3];
console.log(...nums);           // 1 2 3
Math.max(...nums);              // 3

function add(a, b, c) { return a + b + c; }
add(...nums);                   // 6 (1+2+3)
```

**Rest** — собирает значения в массив (в параметрах функции).
**Spread** — раскладывает массив на значения (при вызове).

---

## 8. Возврат значения (`return`)

Функция может вернуть результат через `return`:

```js
function add(a, b) {
  return a + b;
}
const result = add(2, 3);   // 5
```

**Без `return` функция возвращает `undefined`:**
```js
function greet(name) {
  console.log(`Привет, ${name}`);
  // нет return
}
const result = greet("Дана");   // "Привет, Дана" в консоли, result = undefined
```

**Ранний возврат (early return)** — выйти из функции раньше, чем дойдём до конца:

```js
function divide(a, b) {
  if (b === 0) return "Деление на ноль";   // ранний выход
  return a / b;
}
```

---

## 9. Область видимости

Переменные, объявленные **внутри функции**, не видны снаружи:

```js
function f() {
  const secret = 42;
}
f();
console.log(secret);   // ❌ ReferenceError
```

И наоборот — функция видит переменные снаружи:

```js
const global = "Я снаружи";
function f() {
  console.log(global);   // ✅ "Я снаружи"
}
```

## 10. Замыкания (Closures)

**Замыкание** — это когда функция «запоминает» переменные из того места, где она была создана, даже после того, как внешняя функция завершилась.

**Аналогия — рюкзак:** функция кладёт переменные из своего окружения в рюкзак и носит с собой. Куда бы функцию ни передали, она всегда достанет их из рюкзака.

### Пример 1: счётчик

```js
function createCounter() {
  let count = 0;                       // переменная "заперта" внутри

  return {
    increment: function() { count++; return count; },
    decrement: function() { count--; return count; },
    getCount: function() { return count; }
  };
}

const counter = createCounter();
counter.increment();   // 1
counter.increment();   // 2
counter.getCount();    // 2

// count недоступна напрямую!
console.log(counter.count);   // undefined — переменная спрятана
```

Это **инкапсуляция** — счётчик нельзя сломать снаружи, только через предоставленные методы.

### Пример 2: фабрика функций

```js
function multiplier(factor) {
  return function(x) {
    return x * factor;     // «помнит» factor из multiplier
  };
}

const double = multiplier(2);
const triple = multiplier(3);

double(5);    // 10
triple(5);    // 15
```

`double` навсегда запомнил, что `factor === 2`.

### Почему замыкания важны

- **Инкапсуляция** — прятать переменные (как счётчик выше)
- **Фабрики** — создавать функции с настроенным поведением
- **Колбэки и обработчики событий** — функция «помнит» контекст создания
- **Модули** — паттерн модуля построен на замыканиях

## 11. Callback-функции (колбэки)

**Колбэк** — функция, переданная в другую функцию как аргумент, чтобы быть вызванной позже.

```js
function processUser(name, callback) {
  console.log(`Обрабатываю ${name}...`);
  callback(name);
}

processUser("Дана", (name) => {
  console.log(`Готово, ${name}!`);
});
// Обрабатываю Дана...
// Готово, Дана!
```

**Классические примеры колбэков:**
```js
// setTimeout — вызвать через 1 секунду
setTimeout(() => console.log("Прошла секунда"), 1000);

// Метод массива — вызвать для каждого элемента
[1, 2, 3].forEach(n => console.log(n));

// Обработчик клика (в браузере)
button.addEventListener("click", () => console.log("Клик!"));
```

## 12. Рекурсия

**Рекурсия** — функция вызывает сама себя.

```js
function factorial(n) {
  if (n <= 1) return 1;        // базовый случай (когда остановиться)
  return n * factorial(n - 1); // рекурсивный шаг
}
factorial(5);   // 120 (5 * 4 * 3 * 2 * 1)
```

**Как это работает:** каждый вызов создаёт новый кадр в стеке вызовов:
```
factorial(5) → 5 * factorial(4)
             → 5 * 4 * factorial(3)
             → 5 * 4 * 3 * factorial(2)
             → 5 * 4 * 3 * 2 * factorial(1)
             → 5 * 4 * 3 * 2 * 1 = 120
```

**Опасность — переполнение стека:** глубокая рекурсия (>10000 уровней) рушит программу. Для больших задач лучше использовать циклы.

## 13. Чистые функции vs побочные эффекты

**Чистая функция (pure function):**
- При одинаковых входах всегда возвращает одинаковый результат
- Не имеет побочных эффектов (не меняет ничего снаружи)

```js
// Чистая — зависит только от аргументов
function add(a, b) { return a + b; }
```

**Функция с побочным эффектом:**
```js
let total = 0;
function addToTotal(n) {
  total += n;            // побочный эффект: меняет внешнюю переменную
  return total;
}
```

**Почему чистые функции лучше:** их легко тестировать, предсказуемы, не ломают другой код. Старайся писать чистые функции, где возможно.

## 14. Частые ошибки

### Ошибка 1: забыли `return`
```js
function add(a, b) {
  a + b;           // ❌ забыли return → вернёт undefined
}
function add(a, b) {
  return a + b;    // ✅
}
```

### Ошибка 2: стрелочная функция возвращает undefined
```js
const greet = (name) => {
  `Привет, ${name}`;   // ❌ тело в { } → нужен return
};
const greet = (name) => `Привет, ${name}`;   // ✅ без скобок
```

### Ошибка 3: путают declaration и expression
```js
hoisted();                   // ✅ работает (declaration)
function hoisted() { }

notHoisted();                // ❌ ошибка (expression)
const notHoisted = () => { };
```

### Ошибка 4: колбэк забыл скобки
```js
button.addEventListener("click", greet());   // ❌ вызвалась сразу
button.addEventListener("click", greet);      // ✅ передаём функцию
```

### Ошибка 5: стрелочная функция как метод объекта
```js
const obj = {
  name: "Дана",
  greet: () => console.log(this.name),   // ❌ this не указывает на obj
  greetOk() { console.log(this.name) },  // ✅ обычный метод
};
```

## Шпаргалка

```js
// Объявление (всплывает)
function f(a, b) { return a + b; }

// Выражение (не всплывает)
const f = function(a, b) { return a + b; };

// Стрелочная
const f = (a, b) => a + b;
const square = x => x * x;
const greet = () => "Hi";
const makeUser = (name) => ({ name });

// Default parameters
const greet = (name = "Гость") => `Hi, ${name}`;

// Rest
const sum = (...nums) => nums.reduce((a, b) => a + b, 0);

// Spread
const max = Math.max(...[1, 2, 3]);

// Замыкание
const counter = (() => {
  let count = 0;
  return () => ++count;
})();

// Колбэк
setTimeout(() => console.log("tick"), 1000);
[1,2,3].forEach(n => console.log(n));

// Чистая функция
const add = (a, b) => a + b;   // нет побочных эффектов
```
