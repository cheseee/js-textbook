# Глава 3. Условия и ветвления

> **Цель главы:** научиться управлять потоком выполнения программы — выполнять код только при определённых условиях. Освоить `if/else`, `switch`, тернарный оператор и паттерн раннего возврата.

---

## 1. Что такое ветвление

Программа по умолчанию выполняется сверху вниз, строка за строкой. **Ветвление** — это развилка: в зависимости от условия выполняется одна ветка кода или другая.

**Аналогия:** идёшь по тропе, доходишь до развилки. На табличке: «если идёшь направо —海滩, если налево — горы». Ты проверяешь условие (куда хочу) и выбираешь путь.

```js
const age = 20;

if (age >= 18) {
  console.log("Доступ разрешён");
} else {
  console.log("Доступ запрещён");
}
```

## 2. `if / else / else if`

### Базовый `if`

Выполняет блок, если условие truthy.

```js
if (condition) {
  // выполнится, если condition — truthy
}
```

### `if ... else`

Две ветки: одна если да, другая если нет.

```js
if (age >= 18) {
  console.log("взрослый");
} else {
  console.log("ребёнок");
}
```

### `else if` — несколько веток

```js
if (score >= 90) {
  console.log("A");
} else if (score >= 80) {
  console.log("B");
} else if (score >= 70) {
  console.log("C");
} else {
  console.log("F");
}
```

**Важно:** условия проверяются **по порядку сверху вниз**. Как только нашлось первое `true` — соответствующий блок выполняется, остальные пропускаются.

---

## 3. Фигурные скобки

Если блок состоит из **одной инструкции**, скобки можно опустить:

```js
if (age >= 18) console.log("OK");
else console.log("NO");
```

**Но не делай так.** Всегда ставь скобки. Причины:
1. Легко добавить строку позже и забыть скобки → баг
2. Читается хуже
3. Любой линтер (проверка кода) будет ругаться

```js
// ✅ всегда так
if (age >= 18) {
  console.log("OK");
} else {
  console.log("NO");
}
```

---

## 4. Что считается «истинным» в условии

Условие в `if()` не обязано быть строго `true`/`false`. JS приводит любое значение к boolean.

**Falsy (станет `false`):**
```js
false, 0, -0, 0n, "", null, undefined, NaN
```

**Все остальные значения — truthy (станут `true`):**
```js
"0"           // truthy! (непустая строка)
"false"       // truthy! (строка, не boolean)
[]            // truthy! (пустой массив)
{}            // truthy! (пустой объект)
```

**Практические примеры:**
```js
const username = "";       // пустая строка
if (username) {
  console.log("Привет, " + username);   // НЕ выполнится
}

const items = [];          // пустой массив
if (items) {
  console.log("Есть элементы");         // ВЫПОЛНИТСЯ! (массив truthy)
}
if (items.length > 0) {                 // ✅ правильно
  console.log("Есть элементы");
}
```

## 5. Вложенные условия — почему это плохо

```js
// ❌ вложенность — плохой стиль
if (user) {
  if (user.isLoggedIn) {
    if (user.role === "admin") {
      // делать что-то
    }
  }
}
```

Чем глубже вложенность, тем сложнее читать и поддерживать. Это называется «ад вложенности» (nesting hell).

**Решение — ранний возврат** (см. раздел 8) или логические операторы:

```js
// ✅ через && (но не злоупотребляй)
if (user && user.isLoggedIn && user.role === "admin") {
  // делать что-то
}
```

## 6. `switch` — множественный выбор

Когда нужно сравнить одну переменную с разными значениями. Альтернатива длинной цепочке `else if`.

```js
const day = 3;
let dayName;

switch (day) {
  case 1:
    dayName = "Понедельник";
    break;
  case 2:
    dayName = "Вторник";
    break;
  case 3:
    dayName = "Среда";
    break;
  // ...
  default:
    dayName = "Неизвестный день";
}

console.log(dayName);  // "Среда"
```

### `break` — критически важен

Без `break` выполнение «проваливается» в следующий `case` (это называется **fallthrough**):

```js
switch (day) {
  case 1:
    console.log("Пн");
    // забыли break!
  case 2:
    console.log("Вт");   // выведется и для day === 1!
}
```

Иногда fallthrough полезен (несколько case делают одно и то же):
```js
switch (month) {
  case 12:
  case 1:
  case 2:
    console.log("Зима");   // декабрь, январь, февраль → одно действие
    break;
  case 3:
  case 4:
  case 5:
    console.log("Весна");
    break;
}
```

### `switch` vs `if/else`

| | `switch` | `if/else` |
|---|---|---|
| Когда использовать | точное сравнение с конкретными значениями | диапазоны, сложные условия |
| Сравнение | строгое `===` | любое условие |
| Читаемость | лучше для 5+ значений | лучше для 2-3 условий |
| Ошибки | забывают `break` | забывают `=` vs `===` |

**Правило:** `switch` — для конкретных значений («если день === 3»). `if/else` — для условий («если возраст >= 18»).

## 7. Тернарный оператор — повторение с углублением

```js
условие ? значениеЕслиДа : значениеЕслиНет
```

Возвращает значение (в отличие от `if`, который является инструкцией).

```js
const status = age >= 18 ? "взрослый" : "ребёнок";
```

### Когда использовать тернарный
- Нужно значение, а не действие
- Простое условие (A или B)
- В JSX/шаблонах, JSX/шаблонах

```js
// ✅ хорошо
const color = isError ? "red" : "green";
const msg = count === 0 ? "пусто" : `элементов: ${count}`;
```

### Когда НЕ использовать
```js
// ❌ вложенные тернарники — нечитаемо
const result = a > b
  ? (a > c ? a : c)
  : (b > c ? b : c);

// ❌ побочные эффекты
isOK ? doSomething() : doOther();   // лучше через if
```

## 8. Ранний возврат (early return) — главный паттерн

Один из важнейших паттернов для чистого кода. Вместо вложенных `if/else` — проверяем невалидные случаи в начале функции и выходим сразу.

### Было — вложенный ад
```js
function processUser(user) {
  if (user) {
    if (user.isLoggedIn) {
      if (user.age >= 18) {
        if (user.hasSubscription) {
          // основная логика
          return "OK";
        } else {
          return "Нужна подписка";
        }
      } else {
        return "Слишком молод";
      }
    } else {
      return "Не залогинен";
    }
  } else {
    return "Нет пользователя";
  }
}
```

### Стало — ранний возврат
```js
function processUser(user) {
  if (!user) return "Нет пользователя";
  if (!user.isLoggedIn) return "Не залогинен";
  if (user.age < 18) return "Слишком молод";
  if (!user.hasSubscription) return "Нужна подписка";

  // основная логика — на верхнем уровне
  return "OK";
}
```

Разница колоссальная: код плоский, читается сверху вниз, основная логика не утоплена в скобки.

## 9. Практические примеры

### Пример 1: Определение времени суток
```js
function getTimeOfDay(hour) {
  if (hour < 6) return "Ночь";
  if (hour < 12) return "Утро";
  if (hour < 18) return "День";
  return "Вечер";
}
```

### Пример 2: Калькулятор через switch
```js
function calc(a, b, op) {
  switch (op) {
    case "+": return a + b;
    case "-": return a - b;
    case "*": return a * b;
    case "/": return b === 0 ? "Деление на ноль" : a / b;
    default: return "Неизвестная операция";
  }
}
```

### Пример 3: Проверка данных формы
```js
function validateForm(data) {
  if (!data.email) return "Email обязателен";
  if (!data.email.includes("@")) return "Некорректный email";
  if (!data.password) return "Пароль обязателен";
  if (data.password.length < 8) return "Пароль слишком короткий";
  return null;  // null = ошибок нет
}
```

## 10. Частые ошибки

### Ошибка 1: `=` вместо `===`
```js
if (x = 5) { }       // ❌ всегда true (присвоил 5)
if (x === 5) { }     // ✅
```

### Ошибка 2: забыли `break` в switch
```js
switch (x) {
  case 1: doA();   // забыли break
  case 2: doB();   // выполнится и для x === 1!
}
```

### Ошибка 3: проверка пустого массива
```js
if (arr) { }              // всегда true (массив truthy)
if (arr.length > 0) { }   // ✅
if (arr.length) { }       // ✅ (0 falsy)
```

### Ошибка 4: проверка на null
```js
if (x === null) { }       // ✅ правильно
if (typeof x === "null") { }  // ❌ typeof не возвращает "null"
```

### Ошибка 5: слишком много else if
```js
// ❌ если 10+ веток — это уже switch или объект-маппинг
if (status === "active") { }
else if (status === "pending") { }
else if (status === "banned") { }
// ... ещё 7 веток

// ✅ объект-маппинг
const handlers = {
  active: () => { },
  pending: () => { },
  banned: () => { },
};
handlers[status]?.();
```

## Шпаргалка

```js
// Базовый if
if (cond) { } else if (cond2) { } else { }

// switch
switch (val) {
  case 1: doA(); break;
  case 2:
  case 3: doB(); break;   // 2 и 3 → одно действие
  default: doDefault();
}

// Тернарный (короткое условие, вернуть значение)
const x = ok ? "да" : "нет";

// Ранний возврат (главный паттерн)
function f(x) {
  if (!x) return;
  if (x < 0) return;
  // основная логика
}

// Проверка массива/строки
if (arr.length > 0) { }   // не просто if (arr)
if (str !== "") { }       // не просто if (str) если ждёшь непустую

// Объект-маппинг вместо длинного switch
const actions = { a: doA, b: doB };
actions[type]?.();
```
