# Глава 2. Операторы и выражения

> **Цель главы:** разобраться, какие операторы есть в JS, как они работают, и какие подводные камни скрывают. Особенно — `==` vs `===` и логические операторы с коротким замыканием.

---

## 1. Выражение vs оператор

Новички путают эти понятия. Разложим.

**Выражение (expression)** — код, который **вычисляется** до значения.
```js
2 + 2              // выражение → 4
"hello" + " world" // выражение → "hello world"
x > 10             // выражение → true или false
```

**Инструкция (statement)** — код, который **выполняется** как действие.
```js
let x = 5;         // инструкция: создать переменную
if (x > 10) { }    // инструкция: условие
return x;          // инструкция: вернуть значение
```

**Где путаются:** `x = 5` — это инструкция (объявление) И выражение (возвращает 5). Поэтому работает `a = b = 5`. Но в условиях это источник бага:
```js
if (x = 5) { }     // ❌ присвоил 5, и 5 — truthy → всегда true
if (x === 5) { }   // ✅ сравнил
```

## 2. Арифметические операторы

| Оператор | Действие | Пример | Результат |
|----------|----------|--------|-----------|
| `+` | сложение / склейка | `2 + 3` | `5` |
| `-` | вычитание | `5 - 2` | `3` |
| `*` | умножение | `4 * 3` | `12` |
| `/` | деление | `10 / 4` | `2.5` |
| `%` | остаток от деления | `10 % 3` | `1` |
| `**` | возведение в степень | `2 ** 10` | `1024` |

**Важно:** `/` всегда возвращает дробное число, даже если делимое делится нацело. `4 / 2` → `2`, но это число с плавающей точкой, не целочисленный тип.

### Неочевидные нюансы

**Плюс со строками = склейка (конкатенация):**
```js
"2" + 3        // "23" (строка поглотила число)
2 + "3"        // "23"
"2" + "3"      // "23"
```

**Минус/умножение с строками = JS пытается математику:**
```js
"5" - 2        // 3   ("5" → 5)
"5" * "2"      // 10
"abc" - 2      // NaN (не удалось)
```

**Деление на ноль — не ошибка, а Infinity:**
```js
1 / 0          // Infinity
-1 / 0         // -Infinity
0 / 0          // NaN
```

**Унарный плюс превращает строку в число:**
```js
+"42"          // 42
+"abc"         // NaN
+true          // 1
+false         // 0
+null          // 0
```

**Остаток `%` для проверки чётности:**
```js
4 % 2          // 0 → чётное
5 % 2          // 1 → нечётное
10 % 3         // 1
```

## 3. Инкремент и декремент: `++` и `--`

Увеличивают или уменьшают на 1. Бывают префиксные и постфиксные.

### Постфиксный: `i++` — сначала возвращает, потом меняет
```js
let a = 5;
const result = a++;   // result = 5, a = 6
console.log(result);  // 5 (вернул старое значение)
console.log(a);       // 6 (потом увеличил)
```

### Префиксный: `++i` — сначала меняет, потом возвращает
```js
let b = 5;
const result = ++b;   // b = 6, result = 6
console.log(result);  // 6 (сначала увеличил)
console.log(b);       // 6
```

**Наглядная разница:**
```js
let x = 5;
console.log(x++);  // 5 (вернул, потом +1)
console.log(x);    // 6

let y = 5;
console.log(++y);  // 6 (сначала +1, потом вернул)
console.log(y);    // 6
```

**Совет:** в сомнительных случаях не используй `++` в выражениях — выноси на отдельную строку. Так читается понятнее.

## 4. Операторы сравнения: `==` vs `===`

Самый важный раздел главы. Запомни одно правило: **всегда `===`, никогда `==`.** Сейчас поймёшь почему.

| Оператор | Название | Приводит типы? |
|----------|----------|----------------|
| `==` | нестрогое равно | ✅ да (опасно!) |
| `===` | строгое равно | ❌ нет |
| `!=` | не равно | ✅ да |
| `!==` | строго не равно | ❌ нет |
| `>` `<` `>=` `<=` | больше/меньше | зависит |

### Таблица «безумия» `==`

Эти сравнения возвращают `true`. Со `===` они бы вернули `false`.

```js
"" == false         // true  (оба → 0)
"0" == false        // true
0 == false          // true
"0" == 0            // true
null == undefined   // true
[1] == 1            // true  (массив → "1" → 1)
[] == false         // true  (массив → "" → 0)
```

И совершенно безумные:
```js
"\t\n" == 0         // true (пробельные символы → 0)
```

### Несколько `false`, которые многих удивляют
```js
null == 0           // false! (особое правило для null)
NaN == NaN          // false! (NaN не равен ничему)
```

### Почему `===` всегда лучше
- Предсказуемость: сравнивает без сюрпризов
- Безопасность: разные типы всегда дадут `false`
- Читаемость: сразу видишь намерение

**Всегда пиши `===` и `!==`.** Точка.

### `Object.is()` — третье сравнение

Решает два edge-case'а, с которыми `===` не справляется:

```js
Object.is(NaN, NaN)     // true  (NaN === NaN → false)
Object.is(-0, 0)        // false (-0 === 0 → true)
Object.is("a", "a")     // true  (как ===)
```

Используй, когда важно отличить `NaN` или `-0`.

## 5. Логические операторы: `&&`, `||`, `!`

Работают не только с `true`/`false`, но с любыми значениями. Это их главная фишка.

### `!` (НЕ) — переворачивает boolean
```js
!true           // false
!false          // true
!0              // true  (0 — falsy, !0 → true)
!"текст"        // false (непустая строка — truthy)
!!"текст"       // true  (двойное отрицание = приведение к boolean)
```

**Трюк `!!`** — быстрый способ привести к boolean:
```js
!!0             // false
!!"hello"       // true
!!null          // false
!![]            // true
```

### `&&` (И) — возвращает первое falsy
**Короткое замыкание:** идёт слева направо и возвращает первое falsy значение. Если все truthy — возвращает последнее.

```js
true && false        // false
true && true         // true
"hi" && 0            // 0      (0 — falsy, вернул его)
"hi" && "bye"        // "bye"  (оба truthy, вернул последний)
null && "never"      // null   (null — falsy, сразу вернул)
```

**Практический паттерн — защита от undefined:**
```js
const user = { name: "Дана" };
user && user.name           // "Дана"
null && null.name           // null (не упало!)
```

### `||` (ИЛИ) — возвращает первое truthy
Идёт слева направо и возвращает первое truthy значение. Если все falsy — возвращает последнее.

```js
false || true        // true
0 || "default"       // "default"
"" || "fallback"     // "fallback"
null || undefined    // undefined (оба falsy, вернул последний)
```

**Практический паттерн — значение по умолчанию:**
```js
const username = inputName || "Гость";
// если inputName пустой/null/undefined — будет "Гость"
```

### `??` (nullish coalescing) — безопасная альтернатива `||`

Появился в ES2020. Возвращает правую часть, **только если левая — `null` или `undefined`**. В отличие от `||`, не реагирует на `0`, `""`, `false`.

```js
0 || "default"        // "default" (0 — falsy, сработало)
0 ?? "default"        // 0         (0 — не null/undefined, не сработало)
"" || "default"       // "default"
"" ?? "default"       // ""
false || "default"    // "default"
false ?? "default"    // false
```

**Когда что:**
- `||` — для замены любого falsy (0, "", null, undefined)
- `??` — для замены только null/undefined. **Используй `??` по умолчанию**, это безопаснее.

## 6. Операторы присваивания

| Оператор | Эквивалент | Пример |
|----------|------------|--------|
| `=` | присвоить | `x = 5` |
| `+=` | `x = x + y` | `x += 3` |
| `-=` | `x = x - y` | `x -= 3` |
| `*=` | `x = x * y` | `x *= 2` |
| `/=` | `x = x / y` | `x /= 2` |
| `%=` | `x = x % y` | `x %= 3` |
| `**=` | `x = x ** y` | `x **= 2` |
| `&&=` | `x = x && y` | `x &&= y` |
| `\|\|=` | `x = x \|\| y` | `x \|\|= y` |
| `??=` | `x = x ?? y` | `x ??= y` |

---

## 7. Тернарный оператор

Единственный оператор с тремя операндами. Работает как if/else, но возвращает значение.

```js
условие ? значениеЕслиДа : значениеЕслиНет

const status = age >= 18 ? "взрослый" : "ребёнок";
```

**Когда использовать:** для коротких условий, когда нужно значение. Не для сложной логики.

**Когда НЕ использовать:**
```js
// ❌ плохо — сложно читать
const result = a > b ? (a > c ? a : c) : (b > c ? b : c);

// ✅ лучше через if/else или Math.max(a, b, c)
```

## 8. Приоритет операторов

Как в математике: `*` важнее `+`. JS решает порядок по таблице приоритетов.

**Топ приоритетов (от высокого к низкому):**
1. `()` — скобки (всегда побеждают)
2. `.` `[]` — доступ к свойству, вызов функции
3. `!` `++` `--` — унарные
4. `**` — возведение в степень
5. `*` `/` `%` — умножение/деление
6. `+` `-` — сложение/вычитание
7. `<` `>` `<=` `>=` — сравнение
8. `==` `!=` `===` `!==` — равенство
9. `&&` — логическое И
10. `||` `??` — логическое ИЛИ / nullish
11. `? :` — тернарный
12. `=` `+=` `-=` ... — присваивание

**Главный совет:** не пытайся запомнить таблицу. **Используй скобки.** Скобки делают намерение явным и спасают от багов.

```js
// ❌ непонятно без таблицы
2 + 3 * 4 === 14 && !false

// ✅ со скобками — читается
((2 + (3 * 4)) === 14) && (!false)
```

## 9. Частые ошибки

### Ошибка 1: `=` вместо `===` в if
```js
if (x = 5) { }     // ❌ присвоил, всегда true
if (x === 5) { }   // ✅ сравнил
```

### Ошибка 2: `+` со строками и числами
```js
"5" + 2            // "52" (!!!)
Number("5") + 2    // 7
```

### Ошибка 3: NaN не равен NaN
```js
NaN === NaN        // false
Number.isNaN(NaN)  // true (правильно)
```

### Ошибка 4: путают `||` и `??`
```js
0 || "default"     // "default" (0 — falsy)
0 ?? "default"     // 0 (0 — валидное значение!)
```

### Ошибка 5: слишком умный тернарник
```js
a ? b ? c : d : e  // что это вообще значит? разбей на if
```

### Ошибка 6: забывают про приоритет `&&` vs `||`
```js
true || false && false    // true (приоритет && выше)
(true || false) && false  // false
```

### Ошибка 7: `++` в выражениях
```js
arr[i++] = arr[i++] + 1   // что это? не делай так
```

## Шпаргалка

```js
// Сравнение (всегда ===)
5 === "5"                  // false
null == undefined          // true (но не используй ==)
NaN === NaN                // false
Number.isNaN(x)            // проверка на NaN

// Логические
a && b                     // первое falsy или последнее
a || b                     // первое truthy или последнее
a ?? b                     // b если a это null/undefined
!a                         // НЕ
!!a                        // привести к boolean

// Математика
10 % 3                     // 1 (остаток)
2 ** 10                    // 1024 (степень)
+"42"                      // 42 (строку в число)

// Тернарный
const msg = ok ? "да" : "нет";

// Дефолтное значение (предпочитай ??)
const name = input ?? "Гость";
```
